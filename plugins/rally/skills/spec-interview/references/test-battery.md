# 標準試験バッテリー

実装フェーズで**ユーザーに確認せず自動で**展開する機械的試験のカテゴリ表。
インタビューの観点表(checklist.md)が「仕様を確定させる質問」を作るのに対し、
この表は「確定した仕様の周辺を機械的に固める試験」を作る。

## 使い方

- 実装と同時に、対象に該当するカテゴリを選んで E2E(または単体)に展開する。
  全カテゴリ必須ではない — 対象に存在しない概念(永続化しないツールに破損ファイル試験等)は飛ばす
- 試験値は仕様の具体例に合わせる(抽象的な「大きい値」でなく「999999999999999999999」)
- ここにある試験で見つかる問題を人間に触らせて発見させない。人間の役割は
  仕様の解釈ズレ・使い心地という機械化できない反例に限る

## カテゴリ

### 1. 境界値
- 0 件・1 件・ちょうど上限・上限+1
- 期間の端(月末/月初の帰属、年またぎ、うるう年 2/29、平年の 2/29 拒否)
- しきい値ぴったり(予算ちょうど、制限値ちょうど — 超過扱いかを仕様で確認して固定)

### 2. 入力異常系
- 空・空白のみ・null 相当
- 型違い(数値欄に文字)、形式違反(日付 2026-13-01、ゼロ埋めなし 2026-7)
- 過大(桁あふれ級の数値、極端に長い文字列)
- 負数・ゼロ(仕様上の扱いを固定)
- 非 ASCII(日本語・絵文字)と改行・制御文字の混入

### 3. データ永続化の頑健性
- パース不能ファイル(途中で切れた JSON 等)
- **有効だが構造違反**(スキーマ違いの JSON: 型違い・キー欠落・空オブジェクト・リスト)
- 空ファイル(0 バイト)
- 保存先パスの異常(存在しないディレクトリ、ディレクトリを指定)
- **入力パスと出力パスの同一・包含関係**(削除・クリーン等の破壊的処理が入力データを
  巻き込まないか。同一/祖先/子孫のケースを試験し、拒否して何も消さないことを確認)
- 出力先の異常(既存の通常ファイルを出力先に指定、出力先内の読み取り専用ファイル。
  いずれも traceback でなく 1 行エラーで止まること)
- **生成物と入力の名前衝突**(自動生成ファイルと同名の入力があるとき、黙って上書きしない)

### 4. 冪等性・二重実行
- 同じ操作の再実行(二重 done、二重登録)が仕様どおりか(エラー or 冪等)

### 5. 状態遷移の違反
- 存在しない対象への操作、削除済み・完了済みへの再操作
- 順序違反(前提が満たされていない状態での操作)

### 6. 往復一致(ラウンドトリップ)
- 保存→読込→再保存で内容が変わらない
- 表示→(仕様上可能なら)再入力で同じ状態になる

### 7. 不変条件
- エラー終了後にデータファイルが変化していない(before/after 比較)
- 集計の整合(内訳の和=合計)
- 一意性(ID の重複なし・再利用なし)

### 8. 出力契約
- exit code: 成功 0 / 失敗 非 0
- エラーは stderr、正常出力は stdout
- 表示崩れ(巨大値・長い文字列・空一覧)でフォーマットが破綻しない

### 9. 環境依存
- 出力エンコーディング(Windows cp932 環境での非 ASCII 出力)
- 「今日」に依存する処理(日付省略時の既定値)が試験で固定できる作りか
- タイムゾーン・ロケールに依存する処理があれば境界を確認

### 10. スケール smoke(必要な対象のみ)
- 現実的な最大件数(例: 1 万件)で主要操作が現実的な時間・表示で完了する

## 育て方

人間のプローブや後日のバグで「バッテリーに無いカテゴリ」が見つかったら、
一般形に蒸留してこの表へ追記する(checklist.md と同じ規律:
プロジェクト名・日付・経緯は書かない)。
