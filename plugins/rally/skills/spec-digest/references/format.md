# spec.md 固定フォーマット

見出しの番号・名称・順序を変えてはならない。該当が無いセクションは「(なし)」と書いて残す
(セクション自体を消すと版ごとに構成が変わって見えるため)。

```markdown
# <プロジェクト/ツール名> 仕様書

- 生成日: YYYY-MM-DD
- 生成元: tests/e2e/(振る舞い仕様の正)・docs/adr.md
- 注意: 本書は生成物。E2E と食い違う場合は E2E が正。

## 1. 概要

(何をするツールか。3行以内)

## 2. 使い方(コマンド・操作一覧)

(コマンド書式と1行説明の表または一覧)

## 3. 振る舞い仕様

### 3.1 <機能名>

(この機能の確定した振る舞いを Gherkin または「〜すると〜になる」の箇条書きで。
 各項目は E2E で検証されているものだけ)

### 3.2 <機能名>
(機能の数だけサブ節を増やす)

## 4. 入力の制約とエラー時の振る舞い

(拒否される入力と、そのときの挙動・exit code)

## 5. スコープ外(やらないこと)

## 6. 主な設計判断(ADR 要約)

(利用者の理解に効く判断のみ、決定と理由を1行ずつ)

## 7. 未確定・保留事項

(E2E で固定されていない振る舞い。無ければ「(なし)」)
```
